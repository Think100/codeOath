date: 2026-03-20

Markdown is plain text with a thin layer of formatting on top. That is its main advantage.

## What you get

- **Portability.** A `.md` file opens everywhere. No proprietary format, no lock-in.
- **Version control.** Plain text diffs cleanly. You can `git diff` a blog post and actually read the result.
- **Focus.** You write, not click. No toolbar to distract you.

## What you give up

Markdown is not Word. You cannot drag images around a page. Complex tables are awkward. For a blog, none of that matters.

## A code example

```python
def greet(name: str) -> str:
    return f"Hello, {name}"
```

Short, readable, done.

---

For anything where the writing itself is the point, Markdown is hard to beat.
